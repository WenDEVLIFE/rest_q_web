Step 2.2
ML Historical Baseline (Trained Segment Weights)

Before looking at live data, the system queries its ML Historical Baseline — analyzing trained weights for the specific road segment at the current hour. The getTimeMultiplier() encodes known rush-hour capacity: MacArthur Highway at 8:00 AM returns 0.6× speed (60% capacity). The RouteThresholdTuner further self-adjusts these thresholds using historical ground truth samples.

src/service/Traffic_Service.ts TypeScript 
private static getTimeMultiplier(hour: number): number {
  if ((hour >= 7 && hour < 9) || (hour >= 17 && hour < 19)) {
    return 0.60; // Heavy rush — 60% capacity
  } else if (hour >= 11 && hour < 15) {
    return 0.85; // Light mid-day
  } else if (hour >= 19 && hour < 23) {
    return 0.75; // Off-peak
  }
  return 0.95; // Night — near clear
}

Step 2.4 – 2.5
Delta Detection & Hazard Penalty Φ
Historical vs. live divergence triggers weight re-adjustment and 50% speed penalty per verified incident.
TelemetryDriftMonitor.ts Traffic_Service.ts (getIncidentCongestion) XaiBreakdownService.ts (hazardPenalty)

The TelemetryDriftMonitor compares the historical expectation (baseline distribution) against live telemetry using PSI (Population Stability Index) and Jensen-Shannon Divergence. When significant divergence is detected (PSI ≥ 0.25, JSD ≥ 0.06), weights are re-adjusted. A verified incident within 1 km instantly applies a 50% speed reduction (Hazard Penalty Φ). Within 1–2 km it applies a 30% reduction.

src/service/TelemetryDriftMonitor.ts TypeScript 
private static classifySeverity(psi, jsd, maxDelta) {
  if (psi >= 0.50 || jsd >= 0.12 || maxDelta >= 0.28)
    return 'high';    // Major drift — hard re-weight
  if (psi >= 0.25 || jsd >= 0.06 || maxDelta >= 0.18)
    return 'moderate'; // Significant — adjust weights
  if (psi >= 0.10 || jsd >= 0.03 || maxDelta >= 0.10)
    return 'low';      // Minor drift — monitor only
  return 'none';
}
Step 2.6 – 2.7
TSRE Formula · Final Intelligence Synthesis
Master formula fusing all variables into a high-fidelity, anomaly-corrected response time.
Traffic_Service.ts (calculateTSRE) RouteXAI_Service.ts (buildTelemetry) RouteEtaBreakdownService.ts Traffic_Service.ts (detectAndCorrectEtaAnomaly)

All variables are fused into the master TSRE Formula. Baseline distance time, weighted segment congestion (Σ Wi·Ti), hazard penalties (Φ), and weather coefficients (Ω) produce a unified total ETA. An additional anomaly correction engine Winsorizes statistically impossible results (e.g., ETA/km > 9.5 or ratio > 3.6×) by blending with a bounded upper envelope. The result is a confidence score calibrated against historical ground truth response times.

Formula 2.6 · TSRE Master
T = (D × K) + Σ(Wᵢ × Tᵢ) + Φ + Ω
D×K = Baseline (physical distance × 2.5) Σ(Wᵢ×Tᵢ) = Weighted segment congestion terms Φ = Hazard Penalty (0.8 intersection factor) Ω = Weather / Signal Delay (1.2 factor)

src/service/RiskWeightTrainer.ts TypeScript 
private static getHazardFactor(type: string): number {
  const hazardMap: Record<string, number> = {
    fire:     1.0, // Maximum hazard — life-threatening
    flood:    0.8, // High — infrastructure impact
    accident: 0.6, // Moderate — traffic-blocking
    other:    0.4, // Low — situational
  };
  return hazardMap[type.toLowerCase()] || 0.5;
}

Step 3.5
Multivariate R-Score Formula (1–10 Scale)
Blends historical weight, active severity, and velocity variance into a normalized risk score.
RiskWeightTrainer.ts (calculateRScore) FeatureImportanceReportService.ts XaiBreakdownService.ts (proneAreaSeverityIndex)

All risk indicators are fused by the Multivariate R-Score Formula. Historical hazard weight (α·H), current severity of nearby reports (β·S), and speed variance (γ/V) are blended and normalized into a clean 1–10 scale. The weights (α, β, γ) are trained through linear regression on historical incident data via the RiskWeightTrainer, giving the model a self-improving edge.

Formula 3.5 · Risk Score
R = (α·H + β·S + γ/V) / N
α = Hazard factor weight (default 0.40) β = Segment congestion weight (default 0.35) γ = Response time weight / inverse velocity (default 0.25) N = Normalization — scales output to 1–10
src/service/RiskWeightTrainer.ts TypeScript 
static calculateRScore(
  hazardFactor, segmentCongestion,
  responseTimeMin, weights: OptimizedWeights
): number {
  const { alpha, beta, gamma } = weights;
  // γ inverse: faster response → lower risk contribution
  const rtInv = gamma / (responseTimeMin + 0.1);

  const rawScore = alpha * hazardFactor
                 + beta  * segmentCongestion
                 + rtInv;
  const normalized = rawScore / (alpha + beta + gamma);
  return Math.min(10, Math.max(0, normalized * 10));
}
